1 Head First Stats

1.1 Chapter 1 - Pie Charts

  • Pie charts split data into groups or categories
  • Pie charts show Proportion
  • Slices are proportional to the others
  • The larger the slice the higher the frequency
    • Frequency describes how many there are in a group or interval == count
  • Pie charts are useful for showing the variation in groups
    • when the slices are close to the same size, pie charts suck

1.1.1 Chapter 1 - Bar Charts

  • Bar charts show a greater degree of precision
  • Can be drawn horizontally or vertically
  • Each bar represents a category, length represents the value
  • Vertical Bar Charts
    • categories are on the horizontal axis
    • Frequency or percentage is on the vertical axis
  • Horizontal Bar Charts are opposite
  • Golden Rules
    • Charts that show percentages need to indicate frequency in some manner
    • Charts that use percentage that don't show frequency are to be distrusted
      • You can't work out how the data is representative
  • Multiple data set bar charts
    • Split category bar chart - useful to compare frequencies
    • Segmented bar chart - Show frequencies and percentages

1.1.2 Chapter 1 - Categories vs numbers - Histogram

  • Categorical or qualitative data
    • Categorical data is split into categories that describe quantities or characteristics
      • also called qualitative data
      • Key to remember; this data can not be interpreted as numbers
      • Ex. Breed of Dog, Type of Food
  • Numerical or quantitative data
    • Numerical data is data that derives its meaning from numbers
      • It generally involves measurements or counts
      • Also called quantitative because it deals with quantities
      • Ex. Weight, Time, Length
  • Histograms via Grouped data
    • Histograms are like bar charts but present the data on a continuous numeric scale
      • The area of each bar is proportional to the frequency
      • There can be NO gaps between the bars
    • To make a Histogram
      • look at each interval and determine what the range of values will be
      • Ie. 0-199, 200-399 the intervals are going to be -0.5 to 199.5 to 399.5
      • the height of each bar is the frequency
    • Histogram Bar Area - unequal interval width
      • Needs to be proportional to frequency
        • Step 1: find the bar widths numerically
          HoursFrequencyWidth
          0 - 14,3001
          1 - 36,9002
          3 - 54,9002
          5-102,0005
          10-242,10014
        • Step 2: Find the bar heights
          • area of bar = Frequency of group
          • frequency = Width * Height
          • Height = Frequency / Width
            HoursFrequencyWidthHeight
            0 - 14,30014,300 / 1 = 4,300
            1 - 36,90026,900 / 2 = 3,450
            3 - 54,90024,900 / 2 = 2,450
            5-102,00052,000 / 5 = 400
            10-242,100142,100 / 14 = 150
          • The height is used to measure how concentrated the frequency is
            • Also called the Frequency Density
      • Frequency Density = Bar Height (kind of)
        • Think of a thin glass, the full mark on it wouldn't fill wide glass
        • Frequency Density = Frequency / Group width
      • Histograms are bad at presenting running totals
        • This is like asking; "what is the Frequency of people playing halo 10 hours a day?"

1.2 Chapter 2 - Averages: Mean, Median

1.2.1 stats notation

  • Using variables Sum = X1+X2+X3
  • This \(Sum = x_{1} + x_{2} + x_{3}... x_{n} == \sum{x_{n}}\)

1.2.2 Mean (basic average) == Sum / number of x

  • \(\mu = \frac{\sum{x}}{n}\)
  • Mean can be calculated many different ways
    • mean = Sum * Frequency / Total number of entries
    • \(\mu = \frac{\sum{fx}}{\sum{f}}\)
  • Extreme values are called outliers
    • Outliers can have a bad effect on your \(\mu\) or mean
    • Outliers are extreme high/low values that skew or pull the data right/left
  • Useful when the data is fairly symmetrical

1.2.3 Median - the middle value

  • If there is no middle value take the mean of the two middle numbers
  • Median provides a method to eliminate outliers
  • How to find the median in 3 steps
    • 1. Order by ascending value, n = number of entites
    • 2. If n = odd, the middle value is the median
    • 3. If n = even, the mean of the two middle values is the median
  • Find the mean and median in this table
    age123313233
    Freq.342243
    • This data is misrepresenting as there are two sets here
    • Toddlers
      • Mean = (1*3)+(2*4)+(3*2) / 9 = 1.889 or 2
      • Median = 2
    • Adults
      • Mean = (31*2)+(32*4)+(33*3) / 9 = 32.111 or 32
      • Median = 32
  • Mode - The Value in a set of data with the highest frequency
    • ABSOLUTELY has to be a value in the data set and the most frequent
    • Can have more than one mode; called bimodal
    • The example with toddlers/adults above is bimodal
    • The modal class is the highest frequency group in categorical data
  • Problem: CEO wants to give a raise, either $2000 or 10%
    • Mean Salary = $50000, Median = $20000, Mode = $10000
    • What does a $2000 pay raise do to mean, median and mode?
      • Mean, Median and Mode increase by $2000
    • What happens to mean, median and mode if they are given 10%?
      • The mean is increased 1.1 x 50000 = $55000
      • The median is increased 1.1 x 20000 = $22000
      • The mode "" = $12000
    • What would I prefer if I made the mean? and the mode?
      • If you make the median, you prefer the 10%
      • Everything else is the same

1.3 Chapter 3 - Power Ranges

1.3.1 Ranges and Variation

  • The mean doesn't tell you about the data, it simply shows the middle/center

1.3.2 Range

  • Ranges have an upper and lower bound
  • To Find a Range
    • Subtract the Upper bound from the lower bound
    • Range = upper bound - lower bound
